Class ConvertValue

Very fast string to int, long, float, double, bool conversion which also handles formatted numbers (separators, percent, currency).

Optimizations aim at preventing calls to TryParse as early as possible by performing sanity checks on the string. These speed-ups were battle-tested with a huge CSV file of ~45,000 rows times 45 columns (>18 MB!). It made CSV reading 3-5 times faster compared to brute-force TryParse each type one after another.

GetCustomNumberCulture(String, String, String, String)

Returns a CultureInfo specific to converting strings into number types (int, long, float, double) by specifying just the three main number format items: decimal separator, thousand group separator and currency symbol. It applies this style to Number, Percent and Currency values. Use this if you find that ConvertValue.TryParse() is converting formatted numbers incorrectly.

See also: https://docs.microsoft.com/en-us/bingmaps/rest-services/common-parameters-and-types/supported-culture-codes

TryParse(String, CultureInfo)

Fast string to int, long, float, double, bool conversion. Input value is assumed to be whitespace-trimmed. If that is not guaranteed, call input.Trim() before passing it into this method.

Optimizations are (not necessarily in this order): Int/Float NumberStyles do not allow leading/trailing whitespace Rationale: the converter was built for CSV use-case where whitespace never occurs, or if it does, then it cannot be from a cell with a number format. If it can occur in your use-case, simply call str.Trim() on the string you pass into the method. string length > 28 => return input string Rationale: length 29+ exceeds UInt64 length (18446744073709551615 => 20 chars) PLUS possible sign (1), currency (1), group symbols (6) => 20 + 8 = 28 chars max Even double values fit into 24 chars. string length > 15 => try convert to long/double only Rationale: length 16+ exceeds max length of Int32 including sign, currency, group symbols string length 11-15 => if it contains more than 11 digits, input string is returned Rationale: max number of digits in float.MinValue is 11 (less for others) first 3 chars of string aren't digits => return input string Rationale: any common number format will have a digit in the first three chars (ie "$-3,56") first char is '+' => return input string Rationale: numbers are very uncommonly prefixed with + (it's implicit), much more likely this will be a phone number as in "+01 234-5060708090" string length is 4 or 5 => only in this case call bool.TryParse Rationale: bool.TryParse only considers strings like "TRUE" or "false" as bools, never "0" or "1". input string starts or ends with '%' character => trim '%' and float.TryParse the value, if succeeds returns: value / 100.0f Rationale: the '%' symbol will cause float.TryParse to fail even though it could be a percent value char values are not considered => returns a string with length 1 Rationale: char are rarely used as data type anyway, and where the result is known to be a char, simply call Convert.ToChar(str) on the returned string. not considered: decimal, short, byte, unsigned types
